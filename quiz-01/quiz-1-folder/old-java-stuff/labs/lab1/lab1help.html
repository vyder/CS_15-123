Lab 1 Help File

The Graph class Graph.java as given to you, has a constructor that takes the input file name, and the following methods: 

  • readGraphFile(..) // loads the matrix
  • addEdge(..)          // adds weight at src, dest
  • print(..)                // prints a table form dump of the matrix
You are also given a trivial tester program TrivialGraphTester.java that reads a graph input file graphdata.txt into the graph matrix then prints the matrix out in rough table format.

Your first order of business is to study these 2 files then  compile, execute and observe their output until you thoroughly understand them.

Enhancements you are to  make to above class definition

  • Add try catch everywhere needed (start in readGraphFile) so that you can remove all throws IOException clauses everywhere

Methods you are to add to the Graph class

Add the following private new methods to the Graph class.
  1. private boolean hasEdge( int src, int dest ) // T/F edge here ?
  2. private int inDegree( int node )                  // in degree of single node
  3. private int outDegree( int node )                // out degree of single node
  4. private int degree( int node )                     // degree of a single node (= in+out)
Then to add the following public new methods to the Graph class.
  • public int maxInDegree()        
  • public int maxOutDegree()     
  • public int maxDegree()          
  • public int minInDegree()       
  • public int minOutDegree()    
  • public int minDegree()          
A word about efficiency and OOP:    I will be deducting points (small penalty) if you are grossly inefficient in how you write your methods. I will also deduct (a bit more of a penalty for this ) if you write code to do something that could have been done using the member functions already given to you.  Think about every calculation you do and ask: "Is there something already written for me that will do this?" Being able to recognize and use existing code to do a task is a fundamental OOP skill. I will continue to place a premium on in.

Don't store results of previous computations in arrays or data structures - just call the primitives in a loop to compute your mins , maxes etc.

As you add the above methods you should use the Lab1.java (finished Driver file) to test your methods -- or you can add calls to the trivial driver as you add each  method to your Graph.class file. Or - you could stub out ( write empty but compiling versions of ) all the methods right away and put some trivial code in each so your solution will execute with my final driver file Lab1.java. Either way --  test your solution incrementally - test each new methods as you write it.
